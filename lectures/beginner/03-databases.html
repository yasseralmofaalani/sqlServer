المحاضرة 3: إنشاء وإدارة قواعد البيانات

مقدم الدورة: م. ياسر المفعلاني

  • الصفحة الرئيسية
  • المحتوى

إنشاء قاعدة بيانات

الطريقة 1: استخدام T-SQL

CREATE DATABASE قاعدة_البيانات_الأولى; -- مثال بسيط CREATE DATABASE MyDatabase;

الطريقة 2: إنشاء قاعدة بيانات مع خيارات

CREATE DATABASE MyDatabase ON ( NAME = 'MyDatabase_Data', FILENAME = 'C:\Data\MyDatabase.mdf', SIZE = 10MB, MAXSIZE = 100MB, FILEGROWTH = 5MB ) LOG ON ( NAME = 'MyDatabase_Log', FILENAME = 'C:\Data\MyDatabase.ldf', SIZE = 5MB, MAXSIZE = 50MB, FILEGROWTH = 2MB );

شرح المعاملات:

  • NAME: الاسم المنطقي للملف
  • FILENAME: المسار الفعلي للملف على القرص
  • SIZE: الحجم الابتدائي
  • MAXSIZE: الحد الأقصى للحجم
  • FILEGROWTH: مقدار النمو التلقائي

عرض قواعد البيانات

-- عرض جميع قواعد البيانات SELECT name, database_id, create_date FROM sys.databases; -- عرض معلومات قاعدة بيانات محددة EXEC sp_helpdb 'MyDatabase'; -- عرض حجم قاعدة البيانات EXEC sp_spaceused;

تغيير قاعدة البيانات الحالية

-- استخدام قاعدة بيانات معينة USE MyDatabase; GO -- عرض قاعدة البيانات الحالية SELECT DB_NAME() AS CurrentDatabase;
ملاحظة: GO هو أمر خاص في SSMS يفصل بين مجموعات الأوامر.

تعديل قاعدة البيانات

إضافة ملف بيانات جديد

ALTER DATABASE MyDatabase ADD FILE ( NAME = 'MyDatabase_Data2', FILENAME = 'C:\Data\MyDatabase_Data2.ndf', SIZE = 10MB, MAXSIZE = 100MB, FILEGROWTH = 5MB );

تغيير حجم قاعدة البيانات

ALTER DATABASE MyDatabase MODIFY FILE ( NAME = 'MyDatabase_Data', SIZE = 50MB );

تغيير خيارات قاعدة البيانات

-- تغيير Collation ALTER DATABASE MyDatabase COLLATE Arabic_CI_AS; -- تعيين قاعدة البيانات للقراءة فقط ALTER DATABASE MyDatabase SET READ_ONLY; -- إعادة تعيينها للقراءة والكتابة ALTER DATABASE MyDatabase SET READ_WRITE;

حذف قاعدة بيانات

-- حذف قاعدة بيانات (يجب إغلاق جميع الاتصالات أولاً) DROP DATABASE MyDatabase; -- حذف قاعدة بيانات مع إغلاق الاتصالات ALTER DATABASE MyDatabase SET SINGLE_USER WITH ROLLBACK IMMEDIATE; DROP DATABASE MyDatabase;
تحذير: حذف قاعدة البيانات لا يمكن التراجع عنه! تأكد من وجود نسخة احتياطية.

النسخ الاحتياطي (Backup)

إنشاء نسخة احتياطية كاملة

BACKUP DATABASE MyDatabase TO DISK = 'C:\Backup\MyDatabase.bak' WITH FORMAT, NAME = 'Full Backup of MyDatabase', DESCRIPTION = 'Full backup created on ' + CONVERT(VARCHAR(20), GETDATE());

استعادة قاعدة البيانات

RESTORE DATABASE MyDatabase FROM DISK = 'C:\Backup\MyDatabase.bak' WITH REPLACE;

Filegroups - مجموعات الملفات

Filegroups تساعد في تنظيم ملفات قاعدة البيانات:

-- إنشاء Filegroup جديد ALTER DATABASE MyDatabase ADD FILEGROUP FG_Data; -- إضافة ملف إلى Filegroup ALTER DATABASE MyDatabase ADD FILE ( NAME = 'MyDatabase_Data3', FILENAME = 'C:\Data\MyDatabase_Data3.ndf', SIZE = 10MB ) TO FILEGROUP FG_Data;

معلومات إضافية عن قاعدة البيانات

-- عرض معلومات الملفات SELECT name, physical_name, size * 8 / 1024 AS SizeMB, max_size, growth FROM sys.database_files; -- عرض Filegroups SELECT name, type_desc FROM sys.filegroups; -- عرض خيارات قاعدة البيانات SELECT name, collation_name, recovery_model_desc, compatibility_level FROM sys.databases WHERE name = 'MyDatabase';

أفضل الممارسات

  • استخدم أسماء واضحة ووصفية لقواعد البيانات
  • خطط لحجم قاعدة البيانات مسبقاً
  • استخدم Filegroups لتنظيم الملفات الكبيرة
  • قم بعمل نسخ احتياطية منتظمة
  • راقب نمو قاعدة البيانات
  • استخدم مسارات منفصلة لملفات البيانات والـ Log
نصيحة: ضع ملفات البيانات على قرص مختلف عن ملفات الـ Log لتحسين الأداء.

الخلاصة

  • يمكن إنشاء قاعدة بيانات باستخدام T-SQL أو الواجهة الرسومية
  • يمكن تعديل قاعدة البيانات باستخدام ALTER DATABASE
  • النسخ الاحتياطي مهم جداً لحماية البيانات
  • Filegroups تساعد في إدارة الملفات الكبيرة
