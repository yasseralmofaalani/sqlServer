مقدمة عن ArcGIS

ArcGIS هو نظام معلومات جغرافي متكامل من شركة Esri. يمكن ربط SQL Server مع ArcGIS لاستخدام البيانات الجغرافية المخزنة في SQL Server مباشرة في ArcGIS Desktop و ArcGIS Server.

متطلبات الربط

  • SQL Server 2022 أو أحدث مع دعم Spatial Data Types
  • ArcGIS Desktop 10.0 أو أحدث
  • SQL Server Native Client
  • صلاحيات الوصول إلى قاعدة البيانات

إعداد قاعدة البيانات في SQL Server

قبل الربط مع ArcGIS، يجب إعداد قاعدة البيانات بشكل صحيح:

-- إنشاء قاعدة بيانات للربط مع ArcGIS CREATE DATABASE GIS_Database; USE GIS_Database; -- إنشاء جدول ببيانات جغرافية CREATE TABLE المواقع ( ObjectID INT IDENTITY(1,1) PRIMARY KEY, اسم_الموقع NVARCHAR(100) NOT NULL, Shape GEOGRAPHY NOT NULL, الوصف NVARCHAR(500), تاريخ_الإضافة DATETIME DEFAULT GETDATE() ); -- إنشاء Spatial Index CREATE SPATIAL INDEX SIX_Shape ON المواقع(Shape) USING GEOGRAPHY_GRID WITH ( GRIDS = (LEVEL_1 = MEDIUM, LEVEL_2 = MEDIUM, LEVEL_3 = MEDIUM, LEVEL_4 = MEDIUM), CELLS_PER_OBJECT = 16 );
ملاحظة: ArcGIS يتوقع عمود اسمه "Shape" للبيانات الجغرافية. يمكن استخدام أسماء أخرى لكن يتطلب إعدادات إضافية.

إضافة البيانات من ArcGIS

الطريقة 1: استخدام ArcCatalog

  1. افتح ArcCatalog
  2. انتقل إلى "Database Connections"
  3. انقر "Add Database Connection"
  4. أدخل معلومات الاتصال:
    • Database Platform: SQL Server
    • Instance: اسم الخادم
    • Authentication Type: Database authentication أو Operating system authentication
    • Database: GIS_Database
  5. انقر "OK" للاتصال
  6. اسحب الجدول من قاعدة البيانات إلى الخريطة

الطريقة 2: استخدام ArcMap

  1. افتح ArcMap
  2. انقر "Add Data"
  3. اختر "Database Connections"
  4. أنشئ اتصال جديد أو استخدم اتصال موجود
  5. اختر الجدول المطلوب

تصدير البيانات من ArcGIS إلى SQL Server

-- في ArcGIS، استخدم Feature Class to Feature Class -- أو استخدم Python Script: # مثال Python في ArcGIS # import arcpy # arcpy.FeatureClassToFeatureClass_conversion( # "C:/data/points.shp", # "Database Connections/Connection to SQL Server.sde", # "المواقع" # )

استيراد البيانات من SQL Server إلى ArcGIS

يمكن استيراد البيانات مباشرة من SQL Server:

  1. في ArcMap، انقر "Add Data"
  2. اختر "Database Connections"
  3. اتصل بقاعدة البيانات
  4. اختر الجدول الذي يحتوي على عمود Geography
  5. سيتم عرض البيانات على الخريطة تلقائياً

استخدام ArcGIS Server

لنشر البيانات الجغرافية من SQL Server كخدمة:

  1. أنشئ اتصال قاعدة بيانات في ArcGIS Server
  2. أنشئ Map Service يشير إلى البيانات في SQL Server
  3. انشر الخدمة
  4. يمكن الوصول للخدمة من تطبيقات الويب والجوال

مزامنة البيانات

يمكن استخدام Replication في SQL Server لمزامنة البيانات بين SQL Server و ArcGIS:

-- إعداد Replication EXEC sp_addpublication @publication = 'GIS_Publication', @description = 'Publication for GIS data', @sync_method = 'native';

أفضل الممارسات

  • استخدم عمود اسمه "Shape" للبيانات الجغرافية
  • تأكد من وجود Spatial Index
  • استخدم SRID 4326 (WGS84) كمعيار
  • تأكد من صحة البيانات الجغرافية
  • استخدم ObjectID كـ Primary Key
  • احتفظ بنسخ احتياطية منتظمة
تحذير: تأكد من أن جميع البيانات الجغرافية في نفس SRID قبل الربط مع ArcGIS.

استكشاف الأخطاء

  • تأكد من أن SQL Server Native Client مثبت
  • تحقق من صلاحيات المستخدم
  • تأكد من أن عمود Shape موجود وصحيح
  • تحقق من SRID - يجب أن يكون 4326 أو متوافق
  • راجع Event Viewer في Windows للأخطاء

الخلاصة

  • ArcGIS يمكنه الاتصال مباشرة بـ SQL Server
  • استخدم عمود "Shape" للبيانات الجغرافية
  • تأكد من وجود Spatial Index
  • يمكن استيراد وتصدير البيانات بسهولة
  • ArcGIS Server يمكنه نشر البيانات كخدمات
