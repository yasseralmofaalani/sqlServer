صيغ البيانات الجغرافية

هناك عدة صيغ شائعة للبيانات الجغرافية:

  • WKT (Well-Known Text): صيغة نصية
  • WKB (Well-Known Binary): صيغة ثنائية
  • Shapefile: صيغة Esri
  • GeoJSON: صيغة JSON للبيانات الجغرافية
  • KML/KMZ: صيغة Google Earth

استيراد من WKT

-- استيراد Point من WKT DECLARE @wkt NVARCHAR(MAX) = 'POINT(31.2357 30.0444)'; DECLARE @geography GEOGRAPHY = geography::STGeomFromText(@wkt, 4326); INSERT INTO المواقع (اسم_الموقع, Shape) VALUES ('موقع جديد', @geography);

تصدير إلى WKT

SELECT اسم_الموقع, Shape.STAsText() AS WKT FROM المواقع;

استيراد من WKB

-- استيراد من WKB (Binary) DECLARE @wkb VARBINARY(MAX); -- @wkb يحتوي على البيانات الثنائية DECLARE @geography GEOGRAPHY = geography::STGeomFromWKB(@wkb, 4326);

تصدير إلى WKB

SELECT اسم_الموقع, Shape.STAsBinary() AS WKB FROM المواقع;

استيراد Shapefile

لا يمكن استيراد Shapefile مباشرة في SQL Server. يجب استخدام أدوات خارجية:

  • استخدام ArcGIS أو QGIS لتحويل Shapefile إلى SQL Server
  • استخدام ogr2ogr (أداة سطر أوامر)
  • استخدام مكتبات Python مثل geopandas

مثال باستخدام ogr2ogr:

-- في سطر الأوامر: -- ogr2ogr -f "MSSQLSpatial" "MSSQL:server=localhost;database=GIS_Database;trusted_connection=yes" points.shp -nln المواقع

استيراد GeoJSON

-- GeoJSON يحتاج معالجة خاصة -- يمكن استخدام SQL Server 2016+ مع OPENJSON -- أو استخدام Stored Procedure لتحويل GeoJSON إلى WKT CREATE PROCEDURE ImportGeoJSON @json NVARCHAR(MAX) AS BEGIN -- معالجة JSON وتحويله إلى Geography -- (يتطلب SQL Server 2016+ أو مكتبات خارجية) END;

تصدير إلى GeoJSON

-- يمكن إنشاء GeoJSON من SQL Server SELECT '{"type":"Feature","geometry":' + '{"type":"Point","coordinates":[' + CAST(Shape.Long AS VARCHAR) + ',' + CAST(Shape.Lat AS VARCHAR) + ']},"properties":{"name":"' + اسم_الموقع + '"}}' AS GeoJSON FROM المواقع WHERE Shape.STGeometryType() = 'Point';

استيراد KML

KML يحتاج تحويل أولاً. يمكن استخدام أدوات مثل ogr2ogr:

-- تحويل KML إلى SQL Server -- ogr2ogr -f "MSSQLSpatial" "MSSQL:server=localhost;database=GIS_Database" data.kml

تصدير إلى KML

-- يمكن إنشاء KML من SQL Server SELECT '<Placemark><name>' + اسم_الموقع + '</name>' + '<Point><coordinates>' + CAST(Shape.Long AS VARCHAR) + ',' + CAST(Shape.Lat AS VARCHAR) + '</coordinates></Point></Placemark>' AS KML FROM المواقع;

استخدام Bulk Insert

-- إنشاء جدول مؤقت للاستيراد CREATE TABLE #TempImport ( اسم_الموقع NVARCHAR(100), WKT NVARCHAR(MAX) ); -- استيراد البيانات (من ملف CSV مثلاً) BULK INSERT #TempImport FROM 'C:\data\import.csv' WITH ( FIELDTERMINATOR = ',', ROWTERMINATOR = '\n' ); -- تحويل WKT إلى Geography وإدراجه INSERT INTO المواقع (اسم_الموقع, Shape) SELECT اسم_الموقع, geography::STGeomFromText(WKT, 4326) FROM #TempImport; DROP TABLE #TempImport;

أفضل الممارسات

  • استخدم WKT للبيانات البسيطة
  • استخدم WKB للبيانات الكبيرة (أصغر حجماً)
  • تأكد من SRID عند الاستيراد
  • تحقق من صحة البيانات بعد الاستيراد
  • استخدم Bulk Insert للبيانات الكبيرة
  • احتفظ بنسخة احتياطية قبل الاستيراد الكبير

الخلاصة

  • WKT و WKB هما الصيغ الأساسية المدعومة مباشرة
  • Shapefile و GeoJSON تحتاج أدوات خارجية
  • تأكد من SRID عند الاستيراد
  • استخدم Bulk Insert للبيانات الكبيرة
